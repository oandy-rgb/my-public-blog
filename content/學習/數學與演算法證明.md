感覺數學證明只有四種
1.反證法
2.類似dp的歸納法
3.類似貪心的定理推導
4.類似暴力的枚舉

以下是對應的例子：

---

### 1. 反證法 (Proof by Contradiction)
**例子：證明 $\sqrt{2}$ 是無理數（無理數）**

*   **證明過程：**
    1.  **假設反題：** 假設 $\sqrt{2}$ 是有理數，那麼它可以寫成兩個整數 $a/b$ 的比（$a, b$ 互質）。
    2.  **推導：** 平方後得到 $a^2 = 2b^2$。
    3.  **尋找矛盾：** 這意味著 $a^2$ 是偶數 $\to$ $a$ 是偶數 $\to$ 設 $a=2k$ $\to$ 代回 $4k^2 = 2b^2$ $\to$ $2k^2 = b^2$ $\to$ $b$ 也是偶數。
    4.  **結論：** $a$ 和 $b$ 都是偶數，這與「$a, b$ 互質」的假設**矛盾**。
    5.  **結果：** 假設不成立，所以 $\sqrt{2}$ 是無理數。

*   **CS 演算法對應：** **回溯搜尋 (Backtracking)**。
    *   在寫程式時，反證法就是先假設「這條路徑會到終點」，結果發現不對後改走其他條。

---

### 2. 類似 DP 的歸納法 (Mathematical Induction)
**例子：證明 $1 + 2 + \dots + n = \frac{n(n+1)}{2}$**

*   **證明過程：**
    1.  **Base Case (基礎情況)：** 當 $n=1$ 時，$1 = \frac{1(1+1)}{2}$，成立。
    2.  **Inductive Step (遞推步驟)：** 假設當 $n=k$ 時公式成立（$Sum(k)$ 已知）。
    3.  **推導：** 那麼當 $n=k+1$ 時，$Sum(k+1) = Sum(k) + (k+1)$。
    4.  **代換：** 將假設代入：$\frac{k(k+1)}{2} + (k+1) = \frac{(k+1)(k+2)}{2}$。
    5.  **結果：** 證明了 $k \to k+1$ 的狀態轉移成立。

*   **CS 演算法對應：** **動態規劃 (DP) / 遞迴 (Recursion)**。
    *   $Base Case$ 就是 DP 的初始化（邊界）。
    *   $Step$ 就是狀態轉移方程（$DP[i] = DP[i-1] + \dots$）。

---

### 3. 類似貪心的定理推導 (Direct / Constructive Proof)
**例子：證明「兩個偶數的和一定是偶數」**

*   **證明過程：**
    1.  **定義：** 設 $a = 2k$，$b = 2m$（$k, m$ 為整數）。
    2.  **運算：** $a + b = 2k + 2m = 2(k + m)$。
    3.  **結論：** 因為 $k+m$ 是整數，所以 $2(k+m)$ 符合偶數定義。
    4.  **結果：** 得證。

*   **CS 演算法對應：** **貪心演算法 (Greedy) / 線性流程**。
    * 也就是只要局部有效能推廣到全局，就可以考慮看看用貪心，例如在某些情況下，同樣的金額拿硬幣面額最大的可以拿到最少的數量（不一定對）。

---

### 4. 類似暴力的枚舉 (Exhaustive Proof)
**例子：證明「任意整數 $n \in \{1, 2, 3, 4\}$，$n(n+1)$ 都是偶數」**

*   **證明過程：**
    1.  **情況 1：** $n=1 \to 1\times2=2$ (偶)。
    2.  **情況 2：** $n=2 \to 2\times3=6$ (偶)。
    3.  **情況 3：** $n=3 \to 3\times4=12$ (偶)。
    4.  **情況 4：** $n=4 \to 4\times5=20$ (偶)。
    5.  **結果：** 所有情況都驗證了，得證。

*   **CS 演算法對應：** **Brute Force (暴力枚舉)**。
    *   當解空間有限時，這是最穩妥的方法。
    *   著名的**四色定理**證明也是這種思路的極致版（雖然它要檢查的圖形組合太多，人類手算不完，但計算機可以暴力枚舉所有可能的構圖）。

### 結論

上面那幾種幾乎涵蓋了大多數的證明方法，不過我還有漏鴿巢原理之類的方法，而很多數學難題之所以難，是因為我們不知道該切換到哪種模式：
*   推導不下去？ $\to$ **切換反證法**。
*   有遞歸結構？ $\to$ **切換歸納法**。
*   規則很明確？ $\to$ **切換推導法**。
*   範圍很小？ $\to$ **切換枚舉法**。
